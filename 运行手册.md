# WXY-RPC 运行手册

## 1. 环境要求

建议使用以下环境运行本项目：

- JDK 11
- Maven
- Docker
- Zookeeper 3.7

检查环境：

```bash
java -version
mvn -version
docker --version
```

如果当前终端不是 JDK 11，先执行：

```bash
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
```

再次确认：

```bash
java -version
mvn -version
```

如果使用 JDK 21 编译，可能会因为 Lombok 与 JDK 21 不兼容出现以下错误：

```text
java.lang.NoSuchFieldError: JCTree$JCImport qualid
```

遇到该问题时，切换到 JDK 11 后重新编译即可。

## 2. 启动 Zookeeper

如果还没有创建过 Zookeeper 容器：

```bash
docker run -d --name wxy-zk -p 2181:2181 zookeeper:3.7
```

如果容器已经存在：

```bash
docker start wxy-zk
```

检查容器状态：

```bash
docker ps
```

看到类似下面内容表示 Zookeeper 已经启动成功：

```text
0.0.0.0:2181->2181/tcp   wxy-zk
```

## 3. 检查项目配置

provider 配置文件：

```text
provider/src/main/resources/application.properties
```

建议配置为：

```properties
rpc.server.app-name=provider-1
rpc.server.port=9991
rpc.server.registry=zookeeper
rpc.server.registry-addr=127.0.0.1:2181
rpc.server.transport=netty
logging.level.com.wxy.rpc=info
```

consumer 配置文件：

```text
consumer/src/main/resources/application.yml
```

建议配置为：

```yaml
server:
  port: 8881
rpc:
  client:
    load-balance: random
    serialization: PROTOSTUFF
    transport: netty
    registry: zookeeper
    registry-addr: 127.0.0.1:2181
    timeout: 5000

logging:
  level:
    com.wxy.rpc: info
```

## 4. 编译项目

进入项目根目录：

```bash
cd /home/zx/Desktop/wxy-rpc
```

切换到 JDK 11：

```bash
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
```

编译并安装所有模块：

```bash
mvn clean install -DskipTests
```

编译成功后会看到：

```text
BUILD SUCCESS
```

## 5. 启动 Provider

打开第一个终端：

```bash
cd /home/zx/Desktop/wxy-rpc
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
mvn -f provider/pom.xml spring-boot:run
```

Provider 启动后会完成：

```text
1. 连接 Zookeeper
2. 扫描 @RpcService 服务实现类
3. 注册服务到注册中心
4. 将服务实现对象放入本地缓存
5. 启动 Netty RPC 服务端，监听 9991 端口
```

注意：`9991` 是 RPC 通信端口，不是浏览器访问端口。

## 6. 启动 Consumer

打开第二个终端：

```bash
cd /home/zx/Desktop/wxy-rpc
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
mvn -f consumer/pom.xml spring-boot:run
```

Consumer 启动后会监听 HTTP 端口：

```text
8881
```

Consumer 启动过程中会扫描 `@RpcReference` 字段，并注入远程服务代理对象。

## 7. 测试接口

打开第三个终端：

```bash
curl http://localhost:8881/hello/codex
```

预期返回：

```text
Hello, codex
```

测试用户接口：

```bash
curl http://localhost:8881/user/getUser
```

预期返回类似：

```json
{"username":"hwd","password":"123456","age":25}
```

测试用户列表：

```bash
curl http://localhost:8881/user/getAllUser
```

测试抽象服务：

```bash
curl http://localhost:8881/abstracthello/codex
```

## 8. 启动顺序

完整启动顺序如下：

```text
1. 启动 Zookeeper
2. 编译项目
3. 启动 provider
4. 启动 consumer
5. 访问 consumer 暴露的 HTTP 接口
```

一次调用链路：

```text
curl/browser
-> consumer Controller
-> @RpcReference 远程代理对象
-> Zookeeper 服务发现
-> 负载均衡选择 provider
-> Netty 发送 RPC 请求
-> provider 解码请求
-> 反射调用真实服务实现
-> 返回 RpcResponse
-> consumer 返回 HTTP 响应
```

## 9. 常见问题

### 9.1 Zookeeper 容器名称冲突

如果执行：

```bash
docker run -d --name wxy-zk -p 2181:2181 zookeeper:3.7
```

出现：

```text
Conflict. The container name "/wxy-zk" is already in use
```

说明容器已经创建过，直接启动即可：

```bash
docker start wxy-zk
```

### 9.2 JDK 21 编译失败

如果出现：

```text
java.lang.NoSuchFieldError: JCTree$JCImport qualid
```

切换到 JDK 11：

```bash
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
mvn clean install -DskipTests
```

### 9.3 Maven 启动 provider 报找不到 main class

不要使用：

```bash
mvn -pl provider -am spring-boot:run
```

因为它可能会先在父工程 `wxy-rpc` 上执行 Spring Boot 插件，而父工程没有启动类。

改用：

```bash
mvn -f provider/pom.xml spring-boot:run
```

consumer 同理：

```bash
mvn -f consumer/pom.xml spring-boot:run
```

### 9.4 Consumer 调用失败

优先检查：

```bash
docker ps
```

确认 Zookeeper 正常运行。

再检查 provider 是否已经启动，并且注册中心地址都是：

```text
127.0.0.1:2181
```

最后确认 consumer 访问端口是：

```text
8881
```

不是 provider 的 RPC 端口 `9991`。

## 10. 停止服务

停止 provider 和 consumer：

```text
在对应终端按 Ctrl + C
```

停止 Zookeeper：

```bash
docker stop wxy-zk
```

下次重新启动 Zookeeper：

```bash
docker start wxy-zk
```
